# Benchmark results

Checked-in JSON reports from `run_benchmark.py`, one file per configuration.
Each report is self-describing: it records the git revision, package versions,
platform, CPU model, CUDA device, stream sizes, delivery ratios, throughput,
`matching_frame_id` skew counters, and (with `--trace-latency`) end-to-end
input-to-sink latency percentiles, so runs can be reproduced and compared.

Regenerate a report with, for example:

```bash
python ../run_benchmark.py --backend gpu --main-rate 150 \
  --warmup 5 --duration 30 --trace-latency \
  --json-out gpu_sustained_150hz.json
```

Naming convention: `<backend>_<mode>_<rate>.json` (for example
`cpu_sustained_20hz.json`, `gpu_saturation.json`). Do not commit the 1 GiB
reconstructor or any `.npy` calibration arrays — they are regenerated by
`generate_calibrations.py`.
